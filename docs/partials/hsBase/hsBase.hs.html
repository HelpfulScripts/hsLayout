hs
service in module hsBase

Description

Provides a registry for modules, as well as a simple injection mechanism that has similar usage pattern to angularjs.

This allows modules to be used in environments other than the browser and eliminates dependencies on angular.

Methods

  • invoke(name)

    invokes registered modules and injects required parameters. The invoke function will not be called by its name, but rather, by calling the global hs function. Calling with or without new is equivalent. Usage: 

    hs.register('MyExample', function() {...});
var MyInstance1 = hs('MyExample');  
var MyInstance2 = new hs('MyExample');
    Parameters
    ParamTypeDetails
    nameString

    name of the module

    Returns
    *

    the result of calling the constructor function.

  • register(name, fn)

    Registers new modules in an angular-style fashion. Usage: 

    hs.register('MyModule', function() {...});  
 - or - 
hs.register('MyModule', ['MyOtherModule', function(MyOtherModule) {
   ...
}]);
    Parameters
    ParamTypeDetails
    nameString

    the module's name by which it can be used later.

    fnfunction Array

    constructor function, or an array containing previously registered modules that need to be injected into the constructor function, along with the constructor function as last element of the array.

  • reset()

    Resets the hs module to its initial state. All registered components will be removed.
